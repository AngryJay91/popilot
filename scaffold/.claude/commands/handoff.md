# /handoff - 에이전트 간 인수인계

에이전트 간 작업 인수인계를 표준화된 형식으로 수행합니다.

## 사용법

```
/handoff           # 현재 작업의 핸드오프 문서 생성
/handoff to penny  # Simon → Penny 핸드오프
/handoff to vicky  # Penny → Vicky 핸드오프
/handoff to simon  # Vicky → Simon 핸드오프
/handoff to dev    # Penny → 개발팀 핸드오프
```

---

## 핸드오프 흐름

```
🎯 Simon (PRD 완료)
    ↓
📋 Penny (스프린트 계획)
    ↓
👨‍💻 개발팀 (구현)
    ↓
📋 Penny (개발 완료 확인)
    ↓
📊 Vicky (검증)
    ↓
🎯 Simon (결과 해석 & 다음 가설)
```

---

## 🎯→📋 Simon → Penny (PRD 완료)

`$ARGUMENTS`가 `to penny`이면:

```markdown
## 🎯→📋 핸드오프: [기능명]

### PRD 요약
- **One Question**: [질문]
- **가설**: IF [조건] THEN [결과] BECAUSE [이유]
- **OMTM**: [지표] | Before: [값] | Target: [값]
- **검증 기간**: [시작] ~ [종료]

### Guard Rail
- [지표]: ≤[기준]

### Penny에게 요청
- [ ] 스프린트 계획 작성
- [ ] 작업 분해 (개발팀용)
- [ ] 핸드오프 체크리스트 점검

### 참고 문서
- PRD: [노션 링크]
- 디자인: [피그마 링크]
```

## 📋→👨‍💻 Penny → 개발팀 (스토리)

`$ARGUMENTS`가 `to dev`이면:

```markdown
## 📋 핸드오프: [스토리명]

### 배경
- PRD: [링크]
- 왜 필요한가: [한 줄 설명]

### 요구사항
#### 기능 요구사항
- [ ] FR1: [내용]
- [ ] FR2: [내용]

#### 비기능 요구사항
- [ ] NFR1: [성능/보안 등]

### 완료 조건 (Definition of Done)
- [ ] 기능이 정상 동작한다
- [ ] 에러 케이스가 처리된다
- [ ] 이벤트 로깅이 동작한다
- [ ] QA 테스트 통과

### 이벤트 로깅
| 이벤트명 | 트리거 | 파라미터 |
|----------|--------|----------|
| `event_name` | [액션] | param1, param2 |

### 참고 자료
- 디자인: [피그마 링크]
- API 스펙: [링크]
```

## 📋→📊 Penny → Vicky (검증 요청)

`$ARGUMENTS`가 `to vicky`이면:

```markdown
## 📋→📊 핸드오프: [기능명] 검증 요청

### 개발 완료 내용
- 배포일: [날짜]
- 배포 범위: [전체/일부]

### Vicky에게 요청
- [ ] 베이스라인 데이터 수집 (Before)
- [ ] 이벤트 로깅 정상 확인
- [ ] 측정 기간 후 After 데이터 수집
- [ ] 가설 검증 결과 보고

### 측정 대상 이벤트
| 이벤트명 | 설명 |
|----------|------|
| `event_1` | [설명] |
| `event_2` | [설명] |

### 성공/실패 기준
- 성공: [지표] ≥ [값]
- 실패: [지표] < [값]
```

- 💡 새로운 발견: [내용]

### 다음 가설 제안
- [제안 내용]

## 핸드오프 체크리스트

### PRD → 스프린트 (Simon → Penny)
- [ ] One Question이 명확한가?
- [ ] 가설이 IF/THEN/BECAUSE 형식인가?
- [ ] OMTM이 정량화되었는가?
- [ ] Guard Rail이 설정되었는가?

### 스토리 → 개발 (Penny → Dev)
- [ ] 스펙 문서가 완성되었는가?
- [ ] 디자인이 확정되었는가?
- [ ] 이벤트 로깅이 정의되었는가?
- [ ] 완료 조건(DoD)이 명확한가?

### 개발 → 검증 (Penny → Vicky)
- [ ] 배포가 완료되었는가?
- [ ] 이벤트 로깅이 동작하는가?
- [ ] 베이스라인 데이터가 있는가?
- [ ] 측정 기간이 설정되었는가?

### 검증 → 전략 (Vicky → Simon)
- [ ] Before/After가 명확한가?
- [ ] 통계적 유의성을 검토했는가?
- [ ] Guard Rail을 점검했는가?
- [ ] 학습 포인트가 정리되었는가?

## 자동 핸드오프 제안

작업 완료 시점에 자동으로 핸드오프 제안:

| 상황 | 제안 |
|------|------|
| PRD 작성 완료 | "📋 Penny에게 핸드오프할까요?" |
| 스토리 작성 완료 | "👨‍💻 개발팀에 핸드오프할까요?" |
| 개발 완료 확인 | "📊 Vicky에게 검증 요청할까요?" |
| 검증 완료 | "🎯 Simon에게 결과 전달할까요?" |
