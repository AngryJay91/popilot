# /sprint - 스프린트 관리

스프린트 관련 작업을 수행합니다.

## 사용법

```
/sprint          # 현재 스프린트 대시보드 표시
/sprint new      # 새 스프린트 생성
/sprint archive  # 이전 스프린트 아카이브
/sprint status   # 간단한 상태 확인
```

## 현재 스프린트 번호 확인

`CLAUDE.md`에서 "현재 스프린트: **{번호}**" 패턴을 찾아 번호 추출

## 스프린트 대시보드 (기본) ⭐ NEW

`$ARGUMENTS`가 없으면 **대시보드 자동 생성**:

1. `CLAUDE.md`에서 현재 스프린트 번호(N) 확인
2. `.context/sprints/s{N}/context.md` 로드
3. 📋 Penny로서 대시보드 생성:

```markdown
## 📋 Sprint {N} 대시보드

### 🎯 목표
[One Question 또는 스프린트 목표]

### 📊 KR 진행 현황
| KR | 현재 | 목표 | 진행률 | 상태 |
|----|------|------|--------|------|
| KR1 | 15% | 20% | ███████░░░ 75% | 🟢 |
| KR2 | 18% | 15% | ██████████ 100% | ✅ |
| KR3 | 50% | 100% | █████░░░░░ 50% | 🟡 |

### 📝 작업 현황
| 작업 | 담당 | 상태 | 비고 |
|------|------|------|------|
| [작업1] | 개발팀 | 🔄 진행중 | - |
| [작업2] | 마케팅 | ✅ 완료 | - |
| [작업3] | 개발팀 | ⏳ 대기 | 의존성: [작업1] |

### 🚧 블로커
- [ ] [블로커 내용]

### 📅 일정
- 스프린트 기간: [시작일] ~ [종료일]
- 현재: D-{N}일
```

4. 가능하면 GA4에서 현재 지표값 조회 (📊 Vicky 연계)
5. 리스크/블로커 있으면 강조 표시

### 상태 표시 기준
- ✅ 목표 달성 (≥100%)
- 🟢 순항 중 (≥70%)
- 🟡 주의 필요 (≥40%)
- 🔴 위험 (<40%)

## 새 스프린트 생성 (`new`)

`$ARGUMENTS`가 `new`이면:

1. 현재 스프린트 번호 확인
2. 새 스프린트 폴더 생성 (예: `s52/`)
3. `context.md` 템플릿 생성:
   ```markdown
   # Sprint [N] Context

   > **PRD**: [노션 링크]
   > **기간**: [시작일] ~ [종료일]

   ## 목표
   [스프린트 목표 한 문장]

   ## Key Results
   | KR | Metrics | 현재 | 목표 | 담당 |
   |----|---------|------|------|------|
   | KR1 | | | | |

   ## 주요 작업
   - [ ]

   ## 관련 문서
   | 문서 | 링크 |
   |------|------|
   ```

4. 이전 스프린트에 `results.md` 생성 여부 확인

## 아카이브 (`archive`)

`$ARGUMENTS`가 `archive`이면:

1. 종료된 스프린트 폴더를 `_archive/`로 이동
2. 결과 요약이 있는지 확인
3. 없으면 📊 Vicky에게 결과 보고서 작성 제안

## CLAUDE.md 업데이트

새 스프린트 생성 시:
- `CLAUDE.md`의 "현재 스프린트: **{번호}**" 라인만 업데이트
- 예: `## 현재 스프린트: **51**` → `## 현재 스프린트: **52**`
