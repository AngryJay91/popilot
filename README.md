# create-oscar

Multi-agent PO/PM system scaffold for [Claude Code](https://docs.anthropic.com/en/docs/claude-code).

Oscar is a team of 10 AI agents that help Product Owners and Product Managers with product planning, strategy, data analysis, and sprint management.

## Quick Start

```bash
npx create-oscar my-project
cd my-project
```

Then open Claude Code and type `/start` — the Setup Wizard will guide you through configuration.

## What's Included

- **10 Agent personas** — Oscar (orchestrator), Simon (strategy), Penny (planning), Vicky (validation), Danny (analytics), Rita (research), Derek (dev), Quinn (QA), Ollie (task creation), Sage (advisory)
- **29 Slash commands** — Session management, sprint tools, agent activation, domain loading
- **11 Document templates** — PRD, Epic Spec, Story, Screen Spec, Handoff Checklist, etc.
- **5 Oscar workflows** — Setup, multi-agent coordination, session management, task tracking
- **spec-site** — Vue 3 + Vite interactive spec viewer with scenario-based mockups

## Architecture

```
my-project/
├── CLAUDE.md                       # System instructions
├── .claude/commands/               # 29 slash commands
├── .context/
│   ├── project.yaml                # Project config (generated by Setup)
│   ├── agents/                     # 10 agent personas
│   ├── oscar/workflows/            # Oscar extension modules
│   ├── templates/                  # 11 document templates
│   ├── WORKFLOW.md                 # Workflow guide
│   └── ...                         # sprints, sessions, domains, etc.
└── spec-site/                      # Interactive spec viewer
```

## Options

```
npx create-oscar [target-dir] [options]

Options:
  --skip-spec-site   Skip spec-site scaffold
  --force            Overwrite existing files
```

## Setup Wizard

The Setup Wizard runs automatically when you first type `/start` in Claude Code:

1. **Phase 0**: Project type detection (brownfield/greenfield)
2. **Phase 1**: User interview (name, communication style)
3. **Phase 2**: Deep project interview with Simon (15-20 min)
4. **Phase 3**: Integration configuration (GA4, Notion, DB, etc.)
5. **Phase 4**: Template hydration (`.hbs` → `.md`)
6. **Phase 5**: spec-site initialization
7. **Phase 6**: Domain structure creation
